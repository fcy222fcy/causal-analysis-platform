# 智慧养殖环境异常溯源与因果分析系统 - PPT大纲

---

## 1. 系统背景（2-3页）

### 1.1 行业痛点
- 传统养殖业依赖人工经验，异常发现滞后
- 环境异常（温湿度、氨气）与动物行为关联复杂
- 缺乏系统性的异常溯源分析能力
- 从"检测异常"到"解释异常"的技术缺口

### 1.2 系统定位
- 面向养殖场管理与决策系统
- 实现环境异常事件的溯源分析与因果关系建模
- 支持批处理分析与实时异常触发分析

### 1.3 核心价值
- 多源数据融合：环境传感器 + 声纹 + 视频行为
- 因果推断：从相关性分析升级到因果关系建模
- 可解释性：所有异常必须具备"可解释因果链"

---

## 2. 系统架构（3-4页）

### 2.1 四层架构设计
```
┌─────────────────────────────────────────────────────────────┐
│                    前端层（Vue3）                            │
│   ECharts图表 + D3.js因果图 + 溯源路径展示                  │
├─────────────────────────────────────────────────────────────┤
│                    Spring Boot 后端                          │
├─────────────────────────────────────────────────────────────┤
│  Controller层  │  Service层  │  DAO层  │  异步任务管理       │
├─────────────────────────────────────────────────────────────┤
│              MySQL（分库分表）      │     Redis缓存          │
├─────────────────────────────────────────────────────────────┤
│           Python Flask/FastAPI（因果分析服务）               │
│              DoWhy + PyTorch + scikit-learn                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术栈总览
| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue3 + Vite + ECharts + D3.js | 可视化展示 |
| 后端 | Spring Boot 3.x + MyBatis-Plus | 业务逻辑处理 |
| 安全 | Spring Security + JWT | 用户认证授权 |
| 缓存 | Redis | 高频计算结果缓存 |
| 实时 | WebSocket | 实时告警推送 |
| 算法 | Python + DoWhy + PyTorch | 因果分析核心 |

### 2.3 核心业务流程
1. 数据采集：环境传感器 + 声纹 + 视频
2. 时间对齐：秒级同步
3. 异常检测：滑动窗口统计模型 + 声纹分类
4. 因果建模：相关性分析 → 因果图构建 → 路径推断
5. 溯源输出：异常源定位 + 解释性报告

## 3. 数据库设计（2-3页）

### 3.1 分库分表策略
- 按barn_id分库（每个棚舍一个库）
- 按时间维度分区（月分区）
- 保留5年历史数据

### 3.2 核心表结构（9张表）
| 表名 | 用途 | 核心字段 |
|------|------|----------|
| sys_user | 用户管理 | user_id, username, role, barn_id |
| env_sensor_data | 环境传感器数据 | sensor_id, temperature, humidity, ammonia_level |
| animal_behavior | 动物行为数据 | record_id, behavior_type, confidence_score |
| anomaly_event | 异常事件记录 | event_id, event_type, severity_level |
| causal_relation | 因果关系存储 | cause_variable, effect_variable, causal_strength |
| traceability_report | 溯源报告 | report_id, cause_chain, report_content |
| operation_log | 操作日志 | user_id, operation, ip |
| alarm_record | 告警记录 | alarm_id, alarm_level, status |
| system_config | 系统配置 | config_key, config_value |

### 3.3 索引设计
- 时间字段索引：支持历史数据分析
- 业务字段索引：sensor_id, barn_id, event_type等
- 复合索引：优化高频查询性能

## 4. 核心功能展示（4-5页）

### 4.1 数据接入模块
- REST API数据接收
- 模拟数据生成器（支持批量导入）
- 多源数据融合：环境 + 声纹 + 视频

### 4.2 异常检测模块
- 环境异常：基于滑动窗口统计模型
- 行为异常：声纹分类 + 行为识别
- 异常分级：低/中/高三级告警

### 4.3 因果分析模块（核心）
- 相关性分析：皮尔逊/斯皮尔曼相关系数
- 因果图构建：基于DoWhy框架
- 因果路径推断：最优路径算法
- 异常源定位：多变量因果关系建模

### 4.4 溯源报告模块
- 异常事件时间链路构建
- 原因链解释生成
- 可视化溯源路径展示

### 4.5 预警决策模块
- 阈值配置管理：支持按棚舍配置
- 告警规则：单指标超限 + 组合条件触发
- 告警处理闭环：pending → acknowledged → resolved
- 实时推送：WebSocket告警通知

## 5. 技术实现（3-4页）

### 5.1 后端技术实现
- 分层架构：Controller-Service-DAO
- 异步任务处理：CompletableFuture + 线程池
- 缓存策略：Redis缓存高频因果计算结果
- 安全认证：JWT + Spring Security

### 5.2 Python因果分析服务
- 服务框架：Flask/FastAPI
- 因果推断：DoWhy库
- 深度学习：PyTorch
- 数据处理：pandas + scikit-learn

### 5.3 前端技术实现
- 组件化开发：Vue3 Composition API
- 图表可视化：ECharts统计图表
- 因果图展示：D3.js力导向图
- 实时通信：WebSocket

### 5.4 系统集成
- Java ↔ Python通信：HTTP REST API
- 异步任务管理：避免阻塞主线程
- 数据一致性：分库分表策略
